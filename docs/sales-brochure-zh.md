# SecuMesh 企业级 AI 安全治理网关

## 产品宣传手册

---

## 一、产品定位

SecuMesh 是一款面向企业级客户的 AI 安全治理网关，部署在企业内部应用与大模型服务之间，
帮助企业在不改变现有业务系统架构的前提下，快速建立一层统一的 AI 安全、审计与治理能力。

它的核心价值不是“再做一个大模型”，而是帮助企业把大模型真正安全、可控、可审计地用起来。

一句话理解：

**SecuMesh 让企业在接入任意大模型时，先拥有一层自己的安全控制面。**

---

## 二、企业客户面临的现实问题

随着越来越多企业开始将大模型接入客服、知识库、办公助手、销售助手、研发 Copilot、
流程自动化等场景，新的问题也随之出现：

- 员工或系统在调用模型时，可能将手机号、身份证号、客户资料、商业机密直接发送给外部模型
- 企业很难知道“谁在什么时间，调用了哪个模型，问了什么，模型回了什么”
- 不同业务线、不同团队各自接入模型，接口混乱，难以统一管理
- 模型输出可能包含不合规、不安全或不符合企业价值观的内容
- 提示词注入、越权调用等问题会直接影响企业内部 AI 应用安全
- 企业希望灵活切换模型供应商，但又不想对上层业务系统反复改造

很多企业已经意识到：

**真正阻碍 AI 在企业内部规模化落地的，不是模型能力本身，而是安全治理与可控运营能力。**

---

## 三、SecuMesh 能为企业解决什么问题

SecuMesh 通过在业务系统与大模型之间增加一层“AI 安全中间层”，帮助企业建立统一治理能力。

### 1. 敏感信息防泄露

在请求发送给模型之前，SecuMesh 可对手机号、身份证号等敏感信息进行动态脱敏处理，
避免真实数据直接暴露给外部模型服务。

对企业的价值：

- 降低敏感数据外发风险
- 满足企业内部安全规范要求
- 为未来扩展更多敏感字段治理打下基础

### 2. 统一审计留痕

SecuMesh 对每一次模型调用记录结构化审计信息，包括请求标识、会话标识、模型名称、
状态码、耗时、错误类型、安全发现等关键数据。

对企业的价值：

- 事后可追溯
- 便于风险排查和内部审计
- 帮助企业建立 AI 使用责任链路

### 3. 风险内容拦截与治理

SecuMesh 支持对提示词注入风险进行基础检测，并可对模型输出进行敏感词阻断或替换。

对企业的价值：

- 降低越权提示词风险
- 降低违规输出风险
- 为企业建立最基础的内容安全防线

### 4. 多模型统一接入

SecuMesh 兼容 OpenAI 风格接口，可对接 One API 等聚合层，从而间接支持 OpenAI、
OpenRouter 以及多种主流模型服务。

对企业的价值：

- 上层业务只对接一套接口
- 更容易做模型替换和供应商切换
- 避免业务系统被单一模型供应商深度绑定

### 5. 快速落地，低改造成本

SecuMesh 采用兼容 OpenAI API 的网关设计，业务系统通常只需修改 `BASE_URL`
或网关访问地址，即可接入现有安全治理能力。

对企业的价值：

- 上线周期短
- 对现有系统改动小
- 易于作为企业 AI 中台能力逐步推广

### 价值五：让 AI 项目更容易复制推广

当第一批 AI 项目跑通后，SecuMesh 可以作为统一能力底座，复制到更多业务条线，

## 六、与传统直连模型方式相比，有什么不同

### 传统方式

业务系统直接调用模型平台：

业务系统 -> 模型平台

问题在于：

- 缺乏统一安全治理
- 缺乏统一审计
- 模型切换成本高
- 多系统难以统一管理

### SecuMesh 方式

业务系统先接入 SecuMesh，再由 SecuMesh 转发到模型平台：

业务系统 -> SecuMesh -> One API / 模型平台

带来的变化是：

- 模型调用先经过企业自己的安全层
- 所有流量可统一审计
- 后续可以逐步叠加限流、白名单、策略控制、报表分析等治理能力

## 八、客户采购时最关心的几个问题

### 1. 是否需要改造现有系统？

通常不需要大改。

SecuMesh 兼容 OpenAI 风格接口，绝大多数场景只需将原本的大模型调用地址替换为
SecuMesh 网关地址，并配置相应认证信息即可。

### 2. 是否支持私有化部署？

支持。

SecuMesh 采用适合私有化环境的部署方式，可通过 Docker Compose 进行快速部署，
满足企业对内网部署和环境隔离的需求。

### 3. 是否支持多模型切换？

支持。

SecuMesh 本身作为安全治理层，可与 One API 等模型聚合方案结合，帮助企业灵活管理
不同模型供应商。

### 4. 是否支持审计和留痕？

支持。

SecuMesh 已具备结构化审计能力，可记录模型调用的关键元数据，为企业后续风控、
稽核、运维分析提供基础支撑。

### 5. 是否适合先试点再推广？

非常适合。

SecuMesh 可以先从一个业务场景接入，跑通之后再逐步扩展到更多 AI 应用入口，
形成企业统一 AI 安全网关。
